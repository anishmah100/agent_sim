# Pre-Demo Audit — 2026-06-07

Full pass over environment, agent harnesses, UI, and run logs (runs 6-15).
Method: 4 parallel static code/log audits + live functional verb testing
against a running engine (tools/dev-scripts/verb_audit.py) + targeted
live probes. Every claim below is code- or live-confirmed; where a live
test was blocked, it's noted.

North star (docs/PHASE_SOCIAL_EMERGENCE.md): "~10 agents on screen in
real time exhibiting attack, hidden communication, promises, broken
promises, quests, collaboration, scheming, backstabbing, manipulation,
coalitions, contract enforcement — and a UI that makes this legible at a
glance." Phase-7 exit bar: a watcher confirms "yes, this is interesting
emergence."

Verdict: the social LOOP works (validated runs 13/15: contracts
proposed→accepted→honored-by-pay→completed, plus betrayal, combat,
wealth stratification). But several bugs would make a LIVE VC demo look
broken or dumb. Ranked below.

## BLOCKERS (a live demo would visibly break / look broken)

### BLK-1 — Combat & death are visually invisible
- Engine never sets `current_action` to `attack`/`hit`/`interact` (only
  `move`/`wait`/`""`): combat lives in systems/combat/combat.go +
  action.go scenario routing, none set `e.CurrentAction`. (world.go:763,
  action.go:144 are the only setters.)
- Frontend Animator.ts (death fade, hit flash) and Particles.ts
  (attackHit) are **never instantiated** in PixiApp.tsx.
- Entity removal is immediate: when the engine drops a dead entity,
  EntityLayer.setAll() calls `container.destroy()` → dead agents **pop
  out of existence instantly**. No swing, no flinch, no corpse.
- Impact: the headline "kill / backstab" beats are invisible — damage
  just silently happens and a character vanishes. For a demo built on
  combat + betrayal this is the #1 risk.
- Fix: combat/interact handlers set `CurrentAction="attack"|"hit"` for a
  few ticks (mirror move/wait); instantiate Animator+ParticleLayer; gate
  entity removal through a death tween before destroy. (UI side already
  exists, just unwired.)

### BLK-2 — `eat` is broken for every picked-up item (survival loop dead)
- `handlePickup` (inventory.go:43) stores the **entity id**
  (`spawn_160` / `item_221`) in `extras.inventory`, NOT the item's
  `item:<kind>` sprite. Confirmed live: after a successful pickup,
  inventory = `['spawn_160','spawn_114','spawn_26']`.
- `satietyForItem` / `itemKindString` (vitals_snapshot.go:97,
  inventory.go) resolve kind by stripping `item:` and `#` — neither is
  present in an entity id, so the lookup misses → **`eat` returns
  `not_food`** for anything an agent picks up.
- Impact: agents collect food but cannot consume it → they starve. The
  whole survival-economy pillar is non-functional in the live pipeline
  (it only "worked" in unit tests that pre-seed `item:apple` inventory —
  the exact pass-unit-tests/broken-live trap).
- `equip` almost certainly has the same kind-resolution failure for
  picked-up weapons (needs confirm).
- Live confirm of eat was blocked by BLK-3 (couldn't navigate adjacent);
  bug is code-certain.
- Fix: store the item's `item:<kind>#<uniq>` sprite in inventory on
  pickup (resolve from the item entity's Sprite), OR have all consumers
  resolve entity-id → sprite via entity lookup. Former is cleaner.

### BLK-3 — Pathfinding `no_path`: agents frequently can't reach items/targets
- #1 recurring issue in the logs: ~260 "no_path / pathing failed"
  mentions across runs 6-15, still present in run15 (not improving).
- Reproduced live repeatedly: an agent 6 tiles in a straight line from a
  food item could not close the gap (stuck at dist 6); blocked 3 of my
  live verb tests (whisper/give/pay/attack couldn't get adjacent).
- Root: findPath is cardinal-only BFS capped at 64 tiles; dense building
  areas + occupant-blocked goal tiles leave agents wedged, and they
  burn cycles retrying the same blocked move.
- Impact: agents look stuck/dumb on screen and fail to complete
  adjacency actions (trade/give/attack/pickup) → emergence is throttled
  and the demo looks low-agency.
- Fix options: allow diagonal moves; when the goal tile is
  blocked/occupied, path to nearest free adjacent tile; raise/relax the
  64 cap for the clustered demo area; give move a "step as far as you
  can toward target" fallback instead of all-or-nothing.

## MAJOR

### MAJ-4 — Inventory shows opaque entity-ids everywhere (same root as BLK-2)
Inspector Inventory tab and the LLM's own `render_self` show
`spawn_160 ×1` instead of `apple ×1`. The agent literally cannot tell
what it's carrying → degrades all eat/equip/give/trade decisions and
looks broken in the inspector during a demo. Fixed by the BLK-2 fix.

### MAJ-5 — Social-radius tunings are silently ignored
action.go hardcodes speak=3, shout=15, whisper=`>1`; rules.star declares
speak_radius=8, shout_radius=30, whisper_radius=2, shout_muffle_radius=20
— none are read (`TuningInt` never called). Speech carries ~3 tiles
instead of 8, so clustered agents (vision 12) often can't hear each
other — directly undercuts the emergence story. Fix: read the tunings.

### MAJ-6 — WS reconnect: a server/network drop silently kills an agent forever
client.py `_read_loop` is `async for raw in self._ws`; when the socket
closes for ANY reason, the loop ends, the inbox stops filling, and
`observations()` blocks forever — the agent goes dead with no error. The
keepalive fix only prevents *self-inflicted* ping timeouts; real drops
still orphan the agent. Fix: reconnect loop with backoff, or push a
sentinel so the bot can exit/restart cleanly.

### MAJ-7 — Dead LLM agent loops forever, burning Qwen calls
qwen_focal.run() has no `hp<=0` guard. A dead body keeps receiving
observations and calling the LLM every cycle (up to 90s each), emitting
actions the engine rejects — a money/compute sink and a "zombie thinking"
artifact. Fix: skip/stop the loop when self hp<=0.

### MAJ-8 — `reject_task` promised to the model but unemittable
prompt.py tells the model it can "reject" a contract, but reject_task is
NOT in the grammar or to_action — the model can only accept or ignore.
The prompt is lying to the model; contract richness is halved. Fix: add
reject_task to grammar + ACTION_MENU + to_action (one-liner, mirror
accept_task). (complete_task was already added this way and works.)

### MAJ-9 — Killer never loots corpses (dead behavior)
killer.py LOOTING state has a full handler but no code path ever sets it;
after a kill the FSM jumps straight back to HUNTING, abandoning loot. A
visible "kill → loot" beat never happens. Fix: enter LOOTING when the
target disappears and items are nearby.

## MINOR / POLISH

- MIN-10: ~15 fully-implemented engine verbs are unreachable by the LLM
  (defend, heal, drop, work_for_pay, loot, chop, mine, enter/exit,
  claim_ownership, place_blueprint, advance_construction, …) — missing
  from grammar AND to_action. The resources/property/construction economy
  is invisible to agents. (Scope decision: which to expose for the demo.)
- MIN-11: Idle agents make a one-time spawn-settle jump on connect
  (first observation reports a pre-placement tile), then are stable — NOT
  continuous wander (verified: stationary 14s after settle). Cosmetic.
- MIN-12: Harness "VANISHED" false positives — keys on `max_hp==0` which
  conflates a transient empty mental_state read with a real death (body
  often still alive). Pollutes reports; doesn't affect the sim.
- MIN-13: Event-log contamination across engine restarts — scorer filters
  by `tick>=start_tick`, but tick resets on restart while events.jsonl
  persists, so prior-run events leak in. Mitigated by truncating
  events.jsonl per run (done manually); should be automatic (run-id/offset).
- MIN-14: `is_llm` now correct in /agents, but `last_verb`/`last_speech`
  still come back empty (declared+written but not surfaced) — task #243.
- MIN-15: Hovering a background NPC shows a false "rule" badge + 0/0 HP
  (NPCs aren't in /api/v1/agents). Cosmetic demo noise.
- MIN-16: Inspector still has the low-value "Trace" tab; the unique-to-us
  "Witnesses" tab (#236) and death-scream ripple (#238) are unbuilt.
- MIN-17: intent-memory stores the LLM's reasoning sentence, not the
  chosen target — anti-flip-flop nudge entrenches narration, not action.
- MIN-18: manipulator betrayal metric inflated (proposes then betrays on a
  timer regardless of whether the mark accepted — no contract-status read).
- MIN-19: `_salvage_partial` brace-scans raw text; fragile vs nested
  braces in string fields. Bounded by to_action guards.

## Demo-readiness gap (to north star + VC/researcher demo)
The substrate + social loop exist. To make a LIVE demo land:
1. Make combat/death VISIBLE (BLK-1) — non-negotiable for the betrayal story.
2. Fix the survival loop (BLK-2) and agent navigation (BLK-3) so agents
   look purposeful, not stuck/starving.
3. Surface item names (MAJ-4) and correct speech range (MAJ-5).
4. Robustness for an unattended demo: reconnect (MAJ-6), dead-agent guard
   (MAJ-7).
5. Richer behavior: reject_task (MAJ-8), killer loot (MAJ-9).
6. The showcase itself: a curated scenario + Claude agents (sharper play)
   + the narrator telling the story; a clean one-command bring-up; and a
   2-minute scripted "happy path" that reliably shows form-deal → betray →
   narrator-recaps.
For RESEARCHERS specifically: reproducibility/seeding (D23, deferred) and
benchmark packaging are still open.
