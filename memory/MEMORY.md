# Memory pointer

This project's source of truth is the `docs/` folder in the repo root, not auto-memory. A fresh Claude Code session in this directory reads `CLAUDE.md` (which links into `docs/`) and has full context.

If you find yourself wanting to save a "memory" about this project: ask whether it belongs in `docs/DECISIONS.md` (a decision with rationale), `docs/ROADMAP.md` (a planned milestone), `CLAUDE.md` (a durable principle), or doesn't belong as memory at all.

Auto-memory is for things that are genuinely cross-session and DON'T have a natural home in the repo (e.g. "the maintainer prefers terse responses"). Project knowledge lives in the repo.
